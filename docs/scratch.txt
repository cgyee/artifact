Learnings:
Can't use srcDoc as relative paths don't work, which won't allows us to apply the css file similar what we would do in a real world scenario.
Grouping state into a single object seems to be good idea in this case as we don't need to worry about juggling state between the html and css edits for the editor panel.
We can use the same endpoint for both html and css files, also website hrefs are relative to the url path you are currently on.
Backend will need to be refactored soonish since it's not very clean. A lot of repittion and coupling. Will introduce a lot of bugs. Will add tests => refactor => start TDD going forward.
I have broken the render page. Since react router uses client side routing, so I need to figure out how to have the webpage render the html and css files without using an iframe.
- The solution was to reuse the /project/:id endpoint and to use dangerouslySetInnerHTML to render the html and css files. The css files are applied via a style tag in the html since dangerouslySetInnerHTML doesn't fetch the css files.
